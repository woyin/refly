面板展示

  - 英文：Not enough credits to run
  - 中文：积分不足，无法运行

- CTA：Upgrade
- 点击行为：打开会员引导弹窗

全局实时校验
-
必填 Input 未填写
判定规则
- 存在必填（required = true）的 Input
- 且其当前值为空
面板展示
- 类型：Inputs
- 文案：
  - 英文：Required input not filled: {{input_name}}
  - 中文：必填输入项未填写：{{input_name}}
修复引导
- CTA：Configure
- 点击行为：
  - 打开 Input 配置弹窗
  - 值的下方提示文案：
    - 英文：Please complete this required input before running.
    - 中文：请在运行前将该必填输入项补充完整。

全局实时校验
-
工具未安装 / 不可用


判定规则
- Workflow 中存在某个工具
- 且当前用户：
  - 未授权该工具：需要Oauth授权的工具
  - 授权已过期：Oauth授权过，但是目前授权已过期的工具
  - 工具当前不可用：已被下架的工具；或其他人自己安装的 MCP
面板展示
- 类型：Tools
- 文案：
  - 未安装该工具：
    - 英文：Tool not authorized: {{tool_name}}
    - 中文：工具未授权：{{tool_name}}
  - 授权已过期：
    - 英文：Tool authorization expired: {{tool_name}}
    - 中文：工具授权已过期：{{tool_name}}
  - 工具当前不可用：
    - 英文：Tool is unavailable: {{tool_name}}
    - 中文：工具当前不可用：{{tool_name}}
修复引导
- 未授权该工具：
  - CTA：Connect
  - 点击行为：
    - 跳转至 Composio 授权链路
    - 用户完成授权后：
      - 返回当前工作流
      - 自动刷新全局校验结果
- 授权已过期：
  - CTA：Reconnect
  - 点击行为：
    - 跳转至 Composio 重新授权链路
    - 用户完成重新授权后：
      - 返回当前工作流
      - 自动刷新全局校验结果
- 工具当前不可用：
  - CTA：Remove
  - 点击行为：
    - 唤起二次确认弹窗
      - 标题：
        - 英文：Remove this tool?
        - 中文：确认移除该工具？
      - 描述文案
        - 英文：This tool is unavailable. Removing it will also remove it from the agent.
        - 中文：该工具当前不可用。移除后，将从对应 Agent 中删除该工具。
    - 删除完成后：自动刷新全局校验结果
原有的「工具依赖面板」与该校验合为一个
- 原有的「工具依赖面板」触发时机：当出现以下时机时，自动替换为新的校验面板
  - 画布中存在未安装工具
  - 运行工作流时强制提示
  - 运行 Agent 时强制提示
  - Run from here 时强制提示
  - 运行模板时强制提示

6.3 执行
模块
描述
图示
执行前提
- Schedule 状态为 Active
- 系统到达该 Schedule 的 Next Run Time

执行前校验
-
会员定时任务数量校验
校验规则
- 校验当前账号已启用（Active）的定时任务数量
  - Free 用户：最多 1 个
  - Plus 用户：最多 20 个
异常处理
- 若已超过用户当前会员等级的定时任务上限（ Plus 取消订阅转为 Free ）：
  - 仅继续运行当前在启用状态下的，创建时间最早的 Schedule
  - 其他 Schedule 执行时，自动失败，并生成一条运行失败的 Runlog，并向用户发送【已超过定时任务数量上限】的邮件通知（邮件内容详见下文）


执行前校验
-
积分余额校验
校验规则
- 校验当前账号积分是否足够支持本次 Workflow 执行
异常处理
- 若积分不足：
  - 不再继续本次执行
  - 工作流状态不变
  - 生成一条运行失败的 Runlog
  - 向用户发送【已超过定时任务数量上限】的邮件通知（邮件内容详见下文）

执行结果处理与通知
执行成功
当 Workflow 中所有 Agent 节点均成功执行：
- 本次 Run 状态记录为 Success
- 写入 Run Log
- 向用户发送【定时任务运行成功】的邮件通知（邮件内容详见下文）
执行失败
当任一 Agent 节点执行失败：
- 本次 Run 状态记录为 Failed
- 写入 Run Log
- 向用户发送【定时任务运行失败】的邮件通知（邮件内容详见下文）
- Schedule 状态保持不变（仍为 Enabled），可继续参与下一次调度
